# Бот VKEncrypt (Python) — v4.0

Серверная часть VKEncrypt — VK-бот, который работает как **echo-шифратор** с AES-256-GCM. Получает сообщение от `MY_USER_ID`, расшифровывает (если формат `ENC[key_id:base64]`) или шифрует (если обычный текст), и отправляет результат обратно.

Бот использует **ту же модель ключей**, что и userscript: можно либо загрузить прямые hex-ключи из `.env`, либо дать seed-фразу (PBKDF2-SHA256 → k1..k4), либо добавлять ключи командами в чате.

## Структура

```text
bot/
├── main.py                      # Бот: long-poll, KDF, шифрование/расшифровка, команды
├── gen_key.py                   # Генератор одного 256-битного ключа (hex)
├── pyproject.toml               # Зависимости (pycryptodome, vk-api, python-dotenv)
├── uv.lock                      # Зафиксированные версии (uv)
├── .python-version              # Python >= 3.13
├── .env.example                 # Шаблон .env (режим hex-ключей и/или seed)
├── vkencrypt.service.template   # Шаблон systemd-юнита
├── deploy.sh                    # Установка + systemd + (опционально) сборка userscript
├── keys.json                    # Текущие ключи (создаётся командами, gitignored)
└── README.md
```

## Установка и запуск

```bash
cd bot
sudo ./deploy.sh
```

Скрипт автоматически:

1. Создаст `.venv` (через `uv` или `python3 -m venv`).
2. Установит зависимости.
3. Сгенерирует `.env` с двумя случайными ключами K1, K2.
4. Зарендерит `vkencrypt.service` из шаблона и положит в `/etc/systemd/system/`.
5. Запустит бота через systemd.
6. Соберёт актуальный userscript в `../extension/dist/` (если extension существует).

После `deploy.sh` откройте `.env` и впишите свои `VK_TOKEN` и `MY_USER_ID`:

```bash
sudo systemctl stop vkencrypt
nano .env
sudo systemctl start vkencrypt
```

## Получение VK_TOKEN

1. Создайте сообщество ВКонтакте (или используйте существующее).
2. Управление → Дополнительно → Работа с API → Ключи доступа → Создать ключ.
3. Включите **Long Poll API** в настройках сообщества (Типы событий → Сообщения).

## Получение MY_USER_ID

Откройте [id.vk.com/account](https://id.vk.com/account/#/personal) — там указан ваш числовой ID.

## Режимы загрузки ключей

Бот поддерживает три источника ключей одновременно (если указаны — мерджатся):

| Источник | Что | Когда использовать |
|----------|-----|--------------------|
| `PRE_SHARED_KEY_<ID>` в `.env` | Прямой 64-hex ключ | Старый стиль v3.x. Подходит, если ключи уже сгенерированы и лежат в env. |
| `SEED_PHRASE` в `.env` | Фраза → PBKDF2 → k1..k4 | Когда хочется, чтобы расширение и бот шифровали одной фразой (без обмена hex-ключами). |
| Команды `/setseed`, `/setkey` | Сохраняются в `bot/keys.json` | Когда фразу/ключ знает только пользователь, а не тот, кто деплоил. Удобно на shared-хостинге. |

### Режим 1: hex-ключи в `.env`

```env
PRE_SHARED_KEY_K1=<32-байтный hex>
PRE_SHARED_KEY_K2=<32-байтный hex>
DEFAULT_KEY_ID=k1
```

`DEFAULT_KEY_ID` указывает, каким слотом бот шифрует ответы (по умолчанию `k1`).

### Режим 2: seed-фраза в `.env`

```env
SEED_PHRASE=длинная секретная фраза
DEFAULT_KEY_ID=k1
```

Бот деривит `k1..k4` через PBKDF2-SHA256 / 250 000 итераций / salt `vk-p2p-aes-gcm-v1` — **байт-в-байт** так же, как userscript расширения. Если в userscript'е введена та же фраза, k1..k4 совпадут.

### Режим 3: команды в чате

Команды понимает только `MY_USER_ID`. Все ключи, добавленные командами, сохраняются в `bot/keys.json` (chmod 600, gitignored) и подхватываются при следующем старте.

| Команда | Что делает |
|---------|------------|
| `/setseed <фраза>` | Деривит k1..k4 из фразы, сохраняет в keys.json |
| `/delseed` | Удаляет k1..k4 (но не custom-ключи) |
| `/setkey <id> <64-hex>` | Добавляет пользовательский ключ с именем `<id>` |
| `/delkey <id>` | Удаляет пользовательский ключ |
| `/keys` | Список активных ключей |
| `/help` | Справка |

Если бот стартует без единого ключа — он напишет подсказку в лог, а в чате на любое сообщение ответит «нет ни одного ключа» со ссылкой на `/setseed` и `/setkey`.

## Управление сервисом

```bash
sudo systemctl status vkencrypt         # статус
sudo systemctl restart vkencrypt        # перезапуск (после правки .env)
sudo systemctl stop vkencrypt           # остановить
sudo journalctl -u vkencrypt -f         # логи в реальном времени
```

Удаление:

```bash
sudo systemctl disable vkencrypt
sudo rm /etc/systemd/system/vkencrypt.service
sudo systemctl daemon-reload
```

## Генерация нового ключа

```bash
python3 gen_key.py
# -> <64 hex-символа>
```

Скопируйте вывод в `.env` как `PRE_SHARED_KEY_K3=<значение>` (или используйте `/setkey <id> <hex>` в чате) и перезапустите сервис. После правки `.env` пересоберите userscript в `extension/`.

## Синхронизация с расширением

Чтобы userscript и бот шифровали одинаково, выберите один из путей:

- **Через seed-фразу.** В `.env` бота укажи `SEED_PHRASE=...`, в userscript'е введи ту же фразу — обе стороны получат одинаковые k1..k4.
- **Через прямой ключ.** В userscript'е добавь ключ через «➕ Добавить пользовательский ключ», в боте — через `/setkey <id> <тот же hex>`.
- **Через env + добавление в UI.** Положи ключ в `bot/.env` как `PRE_SHARED_KEY_K1=...` — бот подхватит. В userscript'е добавь тот же ключ через «➕ Добавить пользовательский ключ» с тем же id.

## Криптография

- **Алгоритм:** AES-256-GCM
- **Ключ:** 256 бит (32 байта)
- **KDF:** PBKDF2-SHA256, 250 000 итераций, salt `vk-p2p-aes-gcm-v1`, длина 1024 бит → нарезка 4 × 32 байта = k1..k4
- **IV/Nonce:** 12 байт, новый для каждого сообщения
- **Tag:** 16 байт, проверяется автоматически
- **Формат:** `ENC[key_id:base64( iv ‖ ciphertext ‖ tag )]`

Ключи никогда не передаются в сообщениях — только зашифрованный текст. `SEED_PHRASE` и `keys.json` содержат секреты — **не коммитьте их** (оба уже в `.gitignore`).
