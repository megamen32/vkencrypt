# Расширение VKEncrypt (UserScript)

Браузерная часть VKEncrypt — P2P-шифрование сообщений ВКонтакте через AES-256-GCM.

Что делает userscript:

- **Расшифровывает** входящие сообщения формата `ENC[k1:base64...]` прямо в чате (с переключателем `[шифр]/[текст]`).
- **Шифрует** исходящие сообщения кнопкой 🔒 рядом с полем ввода.
- **Хранит ключи** двумя способами: статические (в `STATIC_KEYS` в коде) и временный (только в памяти, вводится через модалку).
- **Генерирует новые 256-битные ключи** в один клик с автокопированием в буфер обмена.

## Структура файла

```text
extension/
├── vkencrypt.template.js   # Шаблон с дефолтными STATIC_KEYS (безопасно коммитить)
├── build.sh                # Сборка userscript + подстановка ключей из ../bot/.env
├── clean.sh                # Удалить dist/
├── dist/                   # Собранные userscript-файлы (gitignored)
└── README.md
```

## Сборка

```bash
cd extension
./build.sh
```

Готовый файл появится в `dist/vkencrypt_userscript_YYYYMMDD_HHMMSS.js`. **Он содержит ваши ключи в открытом виде — не публикуйте его.**

Если в `../bot/.env` есть переменные `PRE_SHARED_KEY_K1..KN`, build.sh подставит их в `STATIC_KEYS`. Если env нет — файл соберётся с теми ключами, что лежат в шаблоне.

Очистить сборку:

```bash
./clean.sh
```

## Установка

### macOS / Windows / Linux — Tampermonkey (Chrome / Firefox / Edge / Safari)

1. Установите расширение **Tampermonkey** из [Chrome Web Store](https://chromewebstore.google.com/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo) (для Firefox — [addons.mozilla.org](https://addons.mozilla.org/firefox/addon/tampermonkey/)).
2. Откройте собранный `dist/vkencrypt_userscript_*.js` в браузере — Tampermonkey автоматически перехватит файл и покажет страницу установки. Нажмите **Install**.
3. Откройте `https://vk.com` и перейдите в любой чат — в поле ввода появится 🔒 и 🔑.

Альтернатива без Tampermonkey: **Violentmonkey** или **Greasemonkey** (Firefox) — те же шаги.

### iPhone / iPad — Safari + Userscripts

1. Установите бесплатное приложение **Userscripts** из [App Store](https://apps.apple.com/app/userscripts/id1463296397).
2. Откройте **Настройки iOS → Safari → Расширения → Userscripts** и включите его. Дайте доступ к `vk.com` и `m.vk.com`.
3. Скопируйте содержимое собранного `dist/vkencrypt_userscript_*.js`.
4. В приложении Userscripts нажмите **+ → New Script**, вставьте код и сохраните.
5. Откройте `vk.com` в Safari — скрипт подключится автоматически.

> Совет: можно также открыть `.js`-файл через Safari Share → Userscripts → Import.

### Android — Kiwi Browser + Tampermonkey

Kiwi — единственный мобильный браузер, поддерживающий полноценные Chrome-расширения.

1. Установите **Kiwi Browser** из [Google Play](https://play.google.com/store/apps/details?id=com.kiwibrowser.browser).
2. Внутри Kiwi откройте Chrome Web Store и установите **Tampermonkey**.
3. Откройте собранный `dist/vkencrypt_userscript_*.js` — Tampermonkey предложит установку.
4. Откройте `vk.com` — скрипт подключится автоматически.

## Использование

1. Откройте любой чат в ВКонтакте.
2. В поле ввода справа появятся две иконки: 🔒 (зашифровать) и 🔑 (выбрать ключ).
3. Наберите текст → нажмите 🔒 → в поле появится шифр вида `ENC[k1:AAAA...]` → отправьте.
4. Собеседник с тем же ключом увидит расшифрованный текст с переключателем `[шифр]/[текст]`.

### Кнопка 🔑 — меню ключей

- **k1..kN** — статические ключи из `STATIC_KEYS` (то же, что в `bot/.env`).
- **@temp** — временный ключ (только в памяти браузера, исчезает при перезагрузке).
- **+ временный ключ** — ввести 64 hex-символа (32 байта) от собеседника.
- **⚡ сгенерировать новый ключ** — создать 256-битный ключ и положить в `@temp`, автокопирование.
- **удалить временный ключ** — очистить `@temp`.

### Обмен ключами по P2P

1. Собеседник А нажимает ⚡ — ключ появляется в `@temp` и копируется в буфер.
2. А отправляет ключ А-лично-Б-у (любым каналом: личное сообщение в не-ВК, QR-код, голосом).
3. Б нажимает 🔑 → **+ временный ключ** → вставляет ключ → Применить.
4. Оба выбирают `@temp` и шифруют сообщения.

## Безопасность

- Ключи никогда не передаются ВКонтакте — только зашифрованный текст.
- Для каждого сообщения генерируется новый 12-байтный IV (безопасно для GCM).
- GCM-tag (16 байт) автоматически проверяется при расшифровке — повреждение/подмена ловится.
- Временный ключ хранится только в памяти страницы — не переживает F5 и не пишется в localStorage.

## Формат ключей в `bot/.env`

```text
PRE_SHARED_KEY_K1=<32-байтный hex>
PRE_SHARED_KEY_K2=<32-байтный hex>
DEFAULT_KEY_ID=k1
```

Можно добавить сколько угодно ключей (K3, K4...) — расширение и бот подхватят их автоматически. После изменения `.env` пересоберите userscript и перезапустите бота.
