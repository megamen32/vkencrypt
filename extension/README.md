# Расширение VKEncrypt (UserScript) — v4.0

Браузерная часть VKEncrypt — P2P-шифрование сообщений ВКонтакте через AES-256-GCM. Расширение и бот-сервер используют **одну и ту же модель ключей** (seed-фраза + пользовательские ключи), так что ввод ключа в браузере и на сервере совпадают.

## Что делает userscript

- **Расшифровывает** входящие сообщения формата `ENC[k1:base64...]` прямо в чате (с переключателем `[шифр]/[текст]`). Тумблер расшифровки в меню.
- **Шифрует** исходящие сообщения — вручную кнопкой 🔒 или автоматически при отправке (Enter, клик Send). Shift+Enter делает перенос строки.
- **Скрывает** кнопку 🔒 при включённом автошифровании.
- **Опционально** кодирует шифротекст в emoji-алфавит (64 эмодзи + паддинг 🟰 + маркер `emj.`) вместо Base64 — для тех, кому Base64 неудобно копировать/читать.
- **Хранит ключи** тремя способами:
  - `DERIVED_KEYS` (k1–k4) — деривация из seed-фразы через PBKDF2-SHA256 / 250 000 итераций. Сохраняется в Tampermonkey по желанию.
  - `CUSTOM_KEYS` — пользовательские 64-hex ключи, добавляются через UI, сохраняются в Tampermonkey.
  - `TEMP_KEY` — временный ключ, только в памяти страницы.
- **Генерирует новые 256-битные ключи** (⚡) с автокопированием в буфер обмена.

> В v4+ ключи **не** бэйкаются в код. Их вводит сам пользователь — либо seed-фразу, либо hex-ключи. Это позволяет расширению и боту работать на одной фразе без правки исходников.

## Структура файла

```text
extension/
├── vkencrypt.template.js   # Исходник userscript (правим руками, коммитим)
├── vkencrypt.user.js       # Установочный файл — копия template, на него указывают @updateURL/@downloadURL
├── build.sh                # Сборка: копирует template в dist/ и в vkencrypt.user.js
├── clean.sh                # Удалить dist/
├── dist/                   # Собранные снапшоты userscript (gitignored)
└── README.md
```

### Почему два `.js` файла — `template.js` и `user.js`

Это разделение «исходник ↔ артефакт» — у них одна и та же роль, но разные имена.

- **Расширение `.user.js` обязательно для Tampermonkey/Userscripts.** Браузер распознаёт userscript только если URL оканчивается на `.user.js` — иначе вместо страницы установки скачается обычный файл. Поэтому стабильный install-URL должен выглядеть как `…/extension/vkencrypt.user.js`.
- **`vkencrypt.template.js`** — единственный источник правок. Логика, UI, селекторы VK. Безопасно коммитится в git — никаких секретов внутри.
- **`vkencrypt.user.js`** — байт-в-байт копия template после сборки. Лежит в репо, чтобы `https://raw.githubusercontent.com/.../extension/vkencrypt.user.js` всегда отдавал валидный userscript (а не 404) и Tampermonkey мог его найти для автообновления.
- **`build.sh`** в v4+ просто публикует актуальный template: копирует его в `dist/vkencrypt_userscript_<timestamp>.js` (снапшот для архива) и перезаписывает `vkencrypt.user.js` (стабильный install-файл). Никакой инжекции ключей — их в коде больше нет.

Если правишь `vkencrypt.template.js` руками — обнови `vkencrypt.user.js` (`./build.sh` или `cp`), иначе Tampermonkey заберёт старую версию при апдейте.

**Можно ли обойтись одним файлом?** Технически да: переименовать template в `vkencrypt.user.js` и редактировать его. Но тогда:
- нельзя держать в репе «безопасный исходник» и публиковать собранный артефакт из разных путей;
- `build.sh` теряет смысл, и `dist/` со снапшотами перестаёт существовать.

Поэтому держим пару.

## Сборка

```bash
cd extension
./build.sh
```

Скрипт скопирует `vkencrypt.template.js` в `dist/vkencrypt_userscript_YYYYMMDD_HHMMSS.js` (снапшот) и в `vkencrypt.user.js` (стабильный install-файл). Ключей в файле нет — secrets не утекают.

Очистить снапшоты:

```bash
./clean.sh
```

## Установка

### macOS / Windows / Linux — Tampermonkey (Chrome / Firefox / Edge / Safari)

1. Установите расширение **Tampermonkey** из [Chrome Web Store](https://chromewebstore.google.com/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo) (для Firefox — [addons.mozilla.org](https://addons.mozilla.org/firefox/addon/tampermonkey/)).
2. Откройте `extension/vkencrypt.user.js` (или снапшот из `dist/`) в браузере — Tampermonkey автоматически перехватит файл и покажет страницу установки. Нажмите **Install**.
3. Откройте `https://vk.com` и перейдите в любой чат — в поле ввода появится 🔒 и 🔑. Появится модалка настройки seed-фразы.

Альтернатива без Tampermonkey: **Violentmonkey** или **Greasemonkey** (Firefox) — те же шаги.

### iPhone / iPad — Safari + Userscripts

1. Установите бесплатное приложение **Userscripts** из [App Store](https://apps.apple.com/app/userscripts/id1463296397).
2. Откройте **Настройки iOS → Safari → Расширения → Userscripts** и включите его. Дайте доступ к `vk.com` и `m.vk.com`.
3. Скопируйте содержимое `vkencrypt.user.js`.
4. В приложении Userscripts нажмите **+ → New Script**, вставьте код и сохраните.
5. Откройте `vk.com` в Safari — скрипт подключится автоматически.

> Совет: можно также открыть `.js`-файл через Safari Share → Userscripts → Import.

### Android — Kiwi Browser + Tampermonkey

Kiwi — единственный мобильный браузер, поддерживающий полноценные Chrome-расширения.

1. Установите **Kiwi Browser** из [Google Play](https://play.google.com/store/apps/details?id=com.kiwibrowser.browser).
2. Внутри Kiwi откройте Chrome Web Store и установите **Tampermonkey**.
3. Откройте `extension/vkencrypt.user.js` — Tampermonkey предложит установку.
4. Откройте `vk.com` — скрипт подключится автоматически.

## Использование

### Первый запуск

Скрипт **не** вылезает с модалкой сразу после установки. Если ключей ещё нет — кнопки 🔒/🔑 рядом с полем ввода открывают модалку настройки.

Введите секретную фразу (≥ 6 символов, лучше длиннее). Из неё детерминированно сгенерятся ключи `k1..k4` — одинаковые на всех устройствах с той же фразой. **Фраза не сохраняется** — сохраняются только производные ключи. В seed-поле есть кнопка 👁️, чтобы посмотреть ввод.

Можно выбрать:
- «Создать ключи» — производные ключи сохранятся в Tampermonkey (не переживут F5 если снимете галку).
- «Только на эту сессию» — ключи живут до перезагрузки страницы.

### Повседневная работа

1. Откройте любой чат в ВКонтакте.
2. В поле ввода справа появятся две иконки:
   - 🔒 — зашифровать текущее сообщение (прячется при включённом автошифровании).
   - 🔑 — открыть меню ключей и настроек.
3. Наберите текст → нажмите 🔒 → в поле появится шифр вида `ENC[k1:AAAA...]` → отправьте.
4. Собеседник с тем же ключом увидит расшифрованный текст с переключателем `[шифр]/[текст]`.

### Автошифрование

В меню 🔑: «🟢 Автошифрование: включено» — скрипт сам зашифрует сообщение перед отправкой (и при клике Send, и при Enter). Shift+Enter / Ctrl+Enter / Alt+Enter / Cmd+Enter остаются переносом строки. После включения ручной 🔒 прячется.

### Emoji-шифротекст

В меню 🔑: «😀 Emoji-шифротекст: включён» — Base64-символы заменяются на 64 эмодзи (плюс паддинг 🟰 и маркер `emj.` в начале). Удобнее читать глазами, но Base64 надёжнее для копирования и пересылки. Применяется только к новым исходящим.

### Кнопка 🔑 — меню

- **k1 / k2 / k3 / k4** — дерив-ключи из seed-фразы (если seed задан).
- **Другие ключи** — пользовательские 64-hex, добавленные через «➕ Добавить пользовательский ключ».
- **⚡ @temp — временный** — сгенерированный или введённый вручную ключ, только в памяти.
- **🟢 Автошифрование: вкл/выкл** — тумблер.
- **😀/🔤 Emoji-шифротекст: вкл/выкл** — тумблер.
- **👁️/🙈 Расшифровка входящих: вкл/выкл** — тумблер.
- **➕ Добавить пользовательский ключ** — модалка с именем и 64 hex-символами.
- **⚡ Сгенерировать временный ключ** — рандомный 256-бит, копируется в буфер, кладётся в `@temp`.
- **🔄 Сменить seed-фразу k1–k4** — модалка с глазом 👁️.
- **🧹 Удалить временный ключ** — очистить `@temp`.
- **🗑️ Удалить ключ `<имя>`** — для каждого пользовательского ключа.
- **🧨 Сбросить все сохранённые ключи** — очистить и DERIVED, и CUSTOM, открыть seed-модалку заново.

ESC закрывает открытое меню.

### Обмен ключами по P2P

**Способ 1: seed-фраза.** Оба собеседника вводят одну и ту же фразу — у обоих будут одинаковые k1..k4.

**Способ 2: hex-ключ.** Собеседник А нажимает ⚡ — ключ появляется в `@temp` и копируется в буфер. А передаёт ключ Б любым каналом (не-вк, QR, голосом). Б добавляет его через «➕ Добавить пользовательский ключ». Оба выбирают ключ и шифруют сообщения.

## Безопасность

- Ключи никогда не передаются ВКонтакте — только зашифрованный текст.
- Seed-фраза никогда не сохраняется ни в Tampermonkey, ни в localStorage, ни в `vk.com` — только производные ключи.
- Для каждого сообщения генерируется новый 12-байтный IV (безопасно для GCM).
- GCM-tag (16 байт) автоматически проверяется при расшифровке — повреждение/подмена ловится.
- Временный ключ хранится только в памяти страницы — не переживает F5 и не пишется в localStorage.
- KDF: PBKDF2-SHA256, 250 000 итераций, salt `vk-p2p-aes-gcm-v1`. Длинная фраза (>50 бит энтропии) делает перебор бессмысленным.

## Синхронизация с ботом

Бот-сервер (`../bot/`) умеет **тот же** KDF и ту же модель ключей. Чтобы расширение и бот шифровали одинаково, нужно либо:

- Ввести **одну и ту же seed-фразу** в обе стороны (например, через `/setseed` в боте).
- Либо добавить **один и тот же 64-hex ключ** через `/setkey` в боте и через «➕ Добавить пользовательский ключ» в расширении.

См. `../bot/README.md`.
