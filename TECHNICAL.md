# VKEncrypt — техническая документация

> Этот документ — справочник для разработчиков и продвинутых пользователей. Если ты просто хочешь шифровать переписку в ВК — тебе в [README.md](README.md).

---

## Что это

VKEncrypt — end-to-end шифрование переписки ВКонтакте через AES-256-GCM. Состоит из двух частей:

- **`extension/`** — userscript для браузера (Tampermonkey / Userscripts / Kiwi).
- **`bot/`** — VK-бот на Python, работает как echo-шифратор для проверки ключей.

Ключи никогда не передаются ВКонтакте — только шифр вида `ENC[k1:base64(iv ‖ ciphertext ‖ tag)]`.

## Подробно про расширение

### Что умеет

- **🔒 Шифрует** исходящие сообщения — кнопка прямо в поле ввода.
- **🔓 Расшифровывает** входящие прямо в чате, с переключателем `[шифр]/[текст]`.
- **🔑 Управляет ключами** — статические (общие с собеседником) и временные (только в памяти браузера).
- **⚡ Генерирует новые 256-битные ключи** в один клик с автокопированием.
- Работает в **любом чате ВК**: личном, групповом, беседе, в старой и новой версии интерфейса.

### Формат шифра

```text
ENC[k1:base64( iv ‖ ciphertext ‖ tag )]
```

| Поле | Размер | Что это |
|------|--------|---------|
| `k1` | 2-8 байт | ID слота ключа (`k1`..`k4` или `@temp`) |
| `iv` | 12 байт | Случайный nonce, новый для каждого сообщения |
| `ciphertext` | переменный | Зашифрованный AES-256-GCM текст |
| `tag` | 16 байт | GCM-тег целостности (проверяется автоматически) |

Если сообщение повреждено или подменено — получатель увидит `[❌ ошибка]`.

### Криптография

| | |
|---|---|
| Алгоритм | **AES-256-GCM** (Web Crypto API, нативно в браузере) |
| Длина ключа | 256 бит (32 байта, 64 hex-символа) |
| IV/Nonce | 12 байт, новый для каждого сообщения |
| Tag | 16 байт, проверяется при расшифровке |
| Внешние библиотеки | **Нет** — только `crypto.subtle` |

Работает полностью офлайн, ничего никуда не отправляется.

### Кнопки в поле ввода

| Кнопка | Что делает |
|--------|------------|
| 🔒 | Зашифровать набранный текст и подставить шифр в поле |
| 🔑 | Открыть меню ключей (выбор, ввод, генерация, удаление) |

### Меню ключей (🔑)

| Пункт | Назначение |
|-------|------------|
| `k1`, `k2`, `k3`, `k4` | Статические ключи из `STATIC_KEYS` |
| `@temp` | Временный ключ (только в памяти, исчезает при F5) |
| **+ временный ключ** | Модалка для ввода 64 hex-символов (полученных от собеседника) |
| **⚡ сгенерировать новый ключ** | Создаёт 256-битный ключ, кладёт в `@temp`, копирует в буфер |
| **удалить временный ключ** | Очищает `@temp` |

### Обмен ключами по P2P

Без сервера, без ВК, без посредников.

1. **А** нажимает 🔑 → ⚡ сгенерировать новый ключ — ключ появляется в `@temp` и **копируется в буфер обмена**.
2. **А** передаёт ключ **Б** любым каналом вне ВК: Signal, QR-код, голосом, в Telegram, на бумажке.
3. **Б** нажимает 🔑 → **+ временный ключ** → вставляет ключ → **Применить**.
4. Оба выбирают `@temp` в меню и обмениваются шифрованными сообщениями.
5. После F5 у обоих временный ключ исчезает.

### ⚠️ Про дефолтные ключи в `STATIC_KEYS`

В шаблоне `extension/vkencrypt.template.js` уже есть 4 готовых ключа `k1..k4` — **это демо**, чтобы скрипт сразу заработал. Их знает каждый, кто прочитал этот репо.

Перед серьёзным использованием:

```bash
cd bot
python3 gen_key.py    # сгенерирует один hex-ключ
```

Скопируй вывод в `bot/.env` как `PRE_SHARED_KEY_K1`, повтори для K2..K4, затем:

```bash
cd ../extension && ./build.sh
```

Build-скрипт подставит твои ключи в `STATIC_KEYS` автоматически. Останется переустановить userscript в Tampermonkey (Dashboard → Replace → новый файл). Проще всего: нажать [установочную ссылку](https://raw.githubusercontent.com/megamen32/vkencrypt/master/extension/vkencrypt.template.js) ещё раз — Tampermonkey заменит старую версию на новую.

**Альтернатива без правки файлов** — используй временные ключи через 🔑 → ⚡. Это безопаснее для одноразовых сессий.

### Автообновление

В шапке userscript указаны `@updateURL` и `@downloadURL`, указывающие на `extension/vkencrypt.template.js` в master-ветке репозитория. При каждом запуске Tampermonkey/Userscripts проверяет эту ссылку: если `@version` вырос — пользователю предлагается обновиться в один клик. Поддерживать актуальную версию скрипта у всех пользователей можно простым `git push`.

### Версионирование

Бампай `@version` в шапке `vkencrypt.template.js` при любом изменении. Семантическое версионирование:

- **patch** (3.3 → 3.4) — мелкие правки, багфиксы.
- **minor** (3.x → 4.0) — новая фича, обратная совместимость шифра сохраняется.
- **major** (x.y → ...) — ломающие изменения формата `ENC[...]` (не делай без крайней нужды, старые шифры перестанут читаться).

## Как менять ключи

### Способ 1: заменить статические ключи

1. Сгенерируй новые ключи:
   ```bash
   cd bot
   for i in 1 2 3 4; do python3 gen_key.py; done
   ```
2. Открой `bot/.env` и замени `PRE_SHARED_KEY_K1..K4` новыми значениями.
3. Перезапусти бота (если запущен):
   ```bash
   sudo systemctl restart vkencrypt
   ```
4. Пересобери userscript:
   ```bash
   cd ../extension && ./build.sh
   ```
5. **Переустанови** userscript в браузере: нажми [установочную ссылку](https://raw.githubusercontent.com/megamen32/vkencrypt/master/extension/vkencrypt.template.js) ещё раз — Tampermonkey покажет «Replace existing script?», соглашайся.

### Способ 2: добавить новый ключ

1. В `bot/.env` добавь строку:
   ```text
   PRE_SHARED_KEY_K5=<новый 64-hex-символа>
   ```
2. Перезапусти бота, пересобери userscript, переустанови в браузере.
3. В чате нажми 🔑 — в меню появится `k5`.

### Способ 3: одноразовый временный ключ (P2P, без правки файлов)

См. раздел «Обмен ключами по P2P» выше. Самый быстрый и безопасный способ для нового собеседника.

### Способ 4: синхронизировать ключи между устройствами

1. На одном устройстве (где собран userscript) перейди по [установочной ссылке](https://raw.githubusercontent.com/megamen32/vkencrypt/master/extension/vkencrypt.template.js) на втором устройстве.
2. Установи через Userscripts (iPhone) или Tampermonkey (PC/Android).
3. Готово — оба устройства используют одни и те же ключи.

## ВК-бот (опционально)

VKEncrypt-бот — это **echo-сервер с AES-256-GCM**, который проверяет, что твои ключи работают: получает твоё сообщение, расшифровывает (или шифрует) и отправляет обратно. Полезен для отладки, проверки новых ключей и обмена зашифрованными сообщениями «с самим собой».

Бот **не хранит** сообщения и **не видит** открытый текст — работает только с шифром.

### Зачем он нужен

- **Проверить ключи** — отправил `hello`, получил `ENC[k1:...]`. Значит ключ работает.
- **Расшифровать вручную** — если на телефоне нет расширения, можно шифровать через бота.
- **Получить эхо** — отправил шифр боту, он расшифровал и прислал обратно зашифрованным.

### Установка

```bash
cd bot
sudo ./deploy.sh
```

Скрипт автоматически:

- создаст `.venv`,
- установит зависимости (`uv sync` или `pip install`),
- сгенерирует `.env` с двумя случайными ключами K1, K2 (если файла ещё нет),
- зарендерит `vkencrypt.service` и положит в `/etc/systemd/system/`,
- запустит бота,
- **соберёт актуальный userscript** в `../extension/dist/` (чтобы бот и расширение использовали одни и те же ключи).

После `deploy.sh` открой `.env` и впиши свои `VK_TOKEN` и `MY_USER_ID`:

```bash
sudo systemctl stop vkencrypt
nano .env
sudo systemctl start vkencrypt
```

### Где взять VK_TOKEN

1. Создай сообщество ВК (или используй существующее).
2. Управление → Дополнительно → Работа с API → Ключи доступа → Создать ключ.
3. В настройках сообщества включи **Long Poll API** (Типы событий → Сообщения).

### Где взять MY_USER_ID

[id.vk.com/account](https://id.vk.com/account/#/personal) — там указан твой числовой ID.

### Управление

```bash
sudo systemctl status vkencrypt         # статус
sudo systemctl restart vkencrypt        # перезапуск (после правки .env)
sudo journalctl -u vkencrypt -f         # логи в реальном времени
```

### Команды в чате с ботом

- `hello` — бот ответит `ENC[k1:...]`.
- `ENC[k1:...]` — бот расшифрует, зашифрует заново и пришлёт обратно.
- `/keys` — показать список активных ключей.

### Криптография

Та же, что в расширении: **AES-256-GCM**, ключи из `bot/.env`. Подробности — в [bot/README.md](bot/README.md).

## Безопасность

### Что шифруется

- **Только текст** в поле ввода и входящие текстовые сообщения формата `ENC[...]`.
- GCM-тег автоматически ловит любую подмену — повреждённое сообщение не расшифруется.

### Что НЕ шифруется

- **Метаданные**: кому и когда отправлено — ВКонтакте это знает.
- **Стикеры, фото, видео, голосовые, файлы, геолокация** — шифруется только текстовое поле.
- **Push-уведомления**: превью сообщения может показать начало шифра, если в настройках ВК включён показ текста в уведомлениях.
- **Содержимое ленты, стены, комментариев** — скрипт работает только в чатах.

### Про хранение ключей

- **Статические** (`STATIC_KEYS`) — в коде userscript, который лежит в Tampermonkey локально. Никуда не отправляются.
- **Временный** (`@temp`) — только в памяти открытой вкладки `vk.com`. Исчезает при F5, не пишется в localStorage/cookies.
- **В `.env` бота** — на сервере, с правами `chmod 600`. Не попадает в git.

### Если компрометирован

Один ключ скомпрометирован — генерируй новые для всех, пересобирай userscript, переустанавливай у всех собеседников. Старые шифры с этого ключа останутся у того, кто их перехватил.
